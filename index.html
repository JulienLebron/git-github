1. Qu'est-ce que Git ?

Git est un système de contrôle de version distribué, créé par Linus Torvalds en 2005. Il a été conçu spécifiquement pour gérer le développement du noyau Linux, mais il est maintenant utilisé dans de nombreux projets logiciels à travers le monde. Git est open source et est compatible avec la plupart des systèmes d'exploitation.

Un système de contrôle de version est un système qui enregistre les modifications apportées à un ensemble de fichiers au fil du temps, de sorte que vous puissiez récupérer des versions spécifiques plus tard si nécessaire. Il vous permet de suivre l'évolution de votre code, de travailler en collaboration avec d'autres développeurs et de revenir à des versions antérieures en cas de besoin.

Pourquoi Git est-il populaire dans le développement logiciel ?

Git se distingue des systèmes de contrôle de version centralisés en étant distribué. Cela signifie que chaque développeur dispose d'une copie complète de l'historique de développement sur sa propre machine. Cette approche permet de travailler en mode déconnecté et facilite la collaboration avec d'autres développeurs.

  • Git est rapide et performant, ce qui est essentiel pour les projets de grande envergure.
  • Il prend en charge les branches, permettant à plusieurs développeurs de travailler simultanément sur différentes fonctionnalités.
  • Git propose des outils puissants pour gérer les conflits de fusion.
  • Il offre une intégration facile avec des services d'hébergement de code tels que GitHub.

Maintenant que nous comprenons les bases de Git, passons à la configuration de Git sur votre machine. Assurez-vous d'avoir Git installé avant de continuer.

2. Configuration de Git:

Lorsque vous installez Git pour la première fois, il est important de configurer quelques paramètres initiaux pour personnaliser votre expérience d'utilisation. Cela inclut votre nom d'utilisateur, votre adresse e-mail, etc.

Exercice pratique 2:

  1. Installation de Git :
    • Rendez-vous sur le site officiel de Git (https://git-scm.com/) et téléchargez la version appropriée pour votre système d'exploitation.
    • Suivez les instructions d'installation pour installer Git sur votre machine.
  2. Configuration initiale de Git :
  3. Ouvrez un terminal et exécutez la commande suivante pour configurer votre nom d'utilisateur Git : git config --global user.name "Votre Nom".
  4. Exécutez la commande suivante pour configurer votre adresse e-mail Git : git config --global user.email "votre@email.com".
  5. Vous pouvez également configurer d'autres options, telles que la couleur de l'interface utilisateur Git, en utilisant la commande : git config --global option valeur.
  6. Expliquer le fichier .gitconfig :

    Le fichier .gitconfig est un fichier de configuration qui stocke les paramètres spécifiques à votre utilisateur Git. Il est situé dans votre répertoire utilisateur (~/.gitconfig sous Linux/macOS ou C:\Users\NomUtilisateur\.gitconfig sous Windows). Vous pouvez l'ouvrir dans un éditeur de texte pour afficher ou modifier ses contenus.

Maintenant que nous avons configuré Git sur votre machine, passons à la partie 3 de notre cours : les commandes de base de Git.

3. Les commandes de base:

Git dispose d'un ensemble de commandes de base qui vous permettent de gérer les versions de votre code et d'enregistrer vos modifications dans un référentiel Git.

Exercice pratique 3:

  1. Initialiser un référentiel Git :
    • Créez un nouveau répertoire vide sur votre machine pour votre projet.
    • Ouvrez un terminal, accédez au répertoire du projet et exécutez la commande suivante : git init.
    • Cela initialisera un nouveau référentiel Git dans votre répertoire.
  2. Effectuer des commits :
    • Créez un fichier dans votre répertoire, ajoutez-y du contenu et enregistrez les modifications.
    • Exécutez la commande git status pour vérifier l'état de vos fichiers.
    • Utilisez la commande git add nom du fichier pour ajouter les fichiers modifiés à la zone de préparation.
    • Exécutez la commande git commit -m "Message de commit" pour enregistrer les modifications dans un commit.
  3. Voir l'historique des commits :
    • Utilisez la commande git log pour afficher l'historique des commits.
    • Examinez les informations affichées, telles que l'auteur du commit, la date et l'heure, le message de commit, etc.
  4. Comparer les versions et annuler les modifications :
    • Utilisez la commande git diff pour afficher les différences entre les modifications non enregistrées et la dernière version enregistrée.
    • Si vous souhaitez annuler les modifications non enregistrées, utilisez la commande git checkout -- nom du fichier.
  5. Ignorer des fichiers avec le fichier .gitignore :
    • Créez un fichier nommé .gitignore dans votre répertoire de projet.
    • Ajoutez les noms de fichiers ou de dossiers que vous souhaitez ignorer dans ce fichier.
    • Ces fichiers ou dossiers seront exclus du suivi Git.

Félicitations ! Vous avez maintenant une bonne compréhension des commandes de base de Git. Vous savez comment initialiser un référentiel Git, effectuer des commits, consulter l'historique des commits, comparer les versions et ignorer des fichiers à l'aide du fichier .gitignore.

Passons maintenant à la partie 4 de notre cours : les branches et la fusion.

4. Branches et fusion:

Git permet de créer et de gérer des branches, ce qui facilite le développement de fonctionnalités indépendamment les unes des autres. Les branches peuvent être fusionnées pour intégrer les modifications.

Exercice pratique 4:

  1. Comprendre les branches :
    • Utilisez la commande git branch pour afficher la liste des branches dans votre référentiel.
    • La branche actuelle est mise en évidence avec un astérisque (*).
  2. Créer une nouvelle branche :
    • Utilisez la commande git branch nom de la branche pour créer une nouvelle branche.
    • Vous pouvez baser la nouvelle branche sur une branche existante en utilisant la commande git branch nom de la branche branche de base
  3. Basculer entre les branches :
    • Utilisez la commande git checkout nom de la branche pour basculer vers une branche spécifique.
    • Assurez-vous d'enregistrer vos modifications avant de basculer vers une autre branche.
  4. Fusionner les branches :
    • Utilisez la commande git merge branche à fusionner pour fusionner une branche dans la branche actuelle.
    • En cas de conflit de fusion, vous devrez résoudre les conflits manuellement avant de terminer la fusion.

Maintenant que vous avez appris à créer des branches, à basculer entre elles et à les fusionner, passons à la dernière partie de notre cours sur le jour 1 : le travail avec des dépôts distants.

5. Travailler avec des dépôts distants:

GitHub est une plateforme d'hébergement populaire pour les dépôts Git. Elle permet de collaborer avec d'autres développeurs et de partager votre code avec la communauté.

Exercice pratique 5:

  1. Introduction à GitHub :
    • Rendez-vous sur le site de GitHub (https://github.com/) et créez un compte si vous n'en avez pas encore.
    • Familiarisez-vous avec l'interface et les fonctionnalités de GitHub.
  2. Créer un nouveau dépôt sur GitHub :
    • Connectez-vous à votre compte GitHub et créez un nouveau dépôt en suivant les instructions fournies.
    • Vous pouvez choisir de rendre le dépôt public ou privé.
  3. Cloner un dépôt existant :
    • Sur la page du dépôt GitHub, cliquez sur le bouton "Code" et copiez l'URL du dépôt.
    • Utilisez la commande git clone URL du dépôt pour cloner le dépôt sur votre machine.
  4. Pousser et tirer les modifications :
    • Effectuez des modifications locales dans votre référentiel.
    • Utilisez la commande git push pour envoyer vos modifications vers le dépôt distant.
    • Utilisez la commande git pull pour récupérer les dernières modifications du dépôt distant.

Félicitations ! Vous avez maintenant une bonne compréhension des branches, de la fusion et du travail avec des dépôts distants. Vous savez comment créer des branches, basculer entre elles, fusionner les modifications et travailler avec GitHub pour partager votre code.

Fonctionnalités avancées de Git:

Git offre une variété de fonctionnalités avancées qui peuvent améliorer votre flux de travail et votre productivité.

Exercice pratique 3:

  1. Créez un alias pour une commande Git fréquemment utilisée en utilisant la commande : git config --global alias.'alias' 'commande'.
  2. Utilisez la fonctionnalité de réécriture de l'historique des commits pour modifier le message d'un commit précédent en utilisant la commande : git rebase -i HEAD~nombre de commits.
  3. Ajoutez un tag à un commit spécifique en utilisant la commande : git tag 'nom du tag' 'ID du commit'.
